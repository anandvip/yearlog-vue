System Overview

Comprehensive architectural blueprint for YearLog Vue.js conversion

📋 System Overview

YearLog is a minimalist personal memory companion designed as a Progressive Web Application (PWA) that emphasizes privacy, simplicity, and offline-first functionality. The application serves as a digital diary and memory strengthening tool, completely self-contained and deployable to GitHub Pages.

Developer Insight: The current vanilla JavaScript implementation is surprisingly robust, but Vue.js will provide better maintainability, reactivity, and component reusability while preserving the minimalist philosophy.

Business Context

YearLog addresses the need for a private, empathetic digital journaling tool that works entirely offline. Unlike cloud-based solutions, it ensures complete user privacy by storing all data locally in browser localStorage, making it ideal for users who prioritize data sovereignty.

Core Value Propositions

  • Complete Privacy: Zero server dependency, all data stays local
  • Offline-First: Full functionality without internet connection
  • Minimalist Design: Clean, distraction-free interface
  • Memory Enhancement: Features designed to strengthen memory connections
  • Cross-Platform: Works on mobile, tablet, and desktop as PWA
🎯 Architecture Goal: Convert to Vue.js while maintaining the existing minimalist philosophy, adding modern development benefits without compromising user experience or increasing complexity.

🔄 Vue.js Conversion Strategy

The conversion strategy focuses on preserving the current functionality while introducing Vue.js benefits: reactivity, component structure, and better state management through Pinia.

State Management with Pinia

// Main store for entries and app state const useAppStore = defineStore('app', () => { const entries = ref([]); const yearName = ref(''); const searchQuery = ref(''); const isSearchOpen = ref(false); // Computed properties const currentYearEntries = computed(() => { const currentYear = new Date().getFullYear(); return entries.value.filter(entry => new Date(entry.timestamp).getFullYear() === currentYear ); }); // Actions const addEntry = (entry) => { entries.value.unshift(entry); saveToStorage(); }; const updateEntry = (index, field, value) => { if (entries.value[index]) { entries.value[index][field] = value; saveToStorage(); } }; const saveToStorage = () => { localStorage.setItem('yearlog_entries', JSON.stringify(entries.value)); localStorage.setItem('yearlog_year_name', yearName.value); }; return { entries, yearName, searchQuery, isSearchOpen, currentYearEntries, addEntry, updateEntry, saveToStorage }; });

⚙️ Technical Architecture

File Structure Options

Option 1: Single File (GitHub Pages Optimized)

yearlog-vue/ ├── index.html # Complete Vue app in single file ├── manifest.json # PWA manifest ├── sw.js # Service worker (optional) ├── assets/ │ ├── ylg.png # App icon │ └── favicon.ico └── README.md

Option 2: Three File Structure

yearlog-vue/ ├── index.html # HTML structure ├── app.css # All styles ├── app.js # Vue app logic ├── manifest.json └── assets/
⚠️ GitHub Pages Constraint: GitHub Pages serves static files only. Vue CLI builds require a build process. The single-file approach or three-file structure ensures direct compatibility without build steps.

Data Persistence Strategy

Maintain the current localStorage approach but enhance it with Vue reactivity:

// Enhanced localStorage with Vue reactivity const useStorageStore = defineStore('storage', () => { const STORAGE_KEY = 'yearlog_entries_v2'; const YEAR_NAME_KEY = 'yearlog_year_name'; const loadFromStorage = () => { try { const entriesData = localStorage.getItem(STORAGE_KEY); const yearNameData = localStorage.getItem(YEAR_NAME_KEY); return { entries: entriesData ? JSON.parse(entriesData) : [], yearName: yearNameData || new Date().getFullYear().toString() }; } catch (error) { console.error('Storage load error:', error); return { entries: [], yearName: new Date().getFullYear().toString() }; } }; const saveToStorage = (entries, yearName) => { try { localStorage.setItem(STORAGE_KEY, JSON.stringify(entries)); localStorage.setItem(YEAR_NAME_KEY, yearName); return true; } catch (error) { console.error('Storage save error:', error); return false; } }; return { loadFromStorage, saveToStorage }; });

PWA Implementation

Enhance the existing PWA features with Vue integration:

// PWA integration in Vue app const usePWAStore = defineStore('pwa', () => { const deferredPrompt = ref(null); const isInstallable = ref(false); const isInstalled = ref(false); const setupPWA = () => { // Handle install prompt window.addEventListener('beforeinstallprompt', (e) => { e.preventDefault(); deferredPrompt.value = e; isInstallable.value = true; }); // Handle app installed window.addEventListener('appinstalled', () => { deferredPrompt.value = null; isInstallable.value = false; isInstalled.value = true; }); }; const installApp = async () => { if (deferredPrompt.value) { deferredPrompt.value.prompt(); const { outcome } = await deferredPrompt.value.userChoice; deferredPrompt.value = null; isInstallable.value = false; return outcome === 'accepted'; } return false; }; return { isInstallable, isInstalled, setupPWA, installApp }; });

// Composable for responsive design import { ref, onMounted, onUnmounted } from 'vue'; export function useResponsive() { const windowWidth = ref(window.innerWidth); const isMobile = computed(() => windowWidth.value < 768); const isTablet = computed(() => windowWidth.value >= 768 && windowWidth.value < 1024); const isDesktop = computed(() => windowWidth.value >= 1024); const updateWidth = () => { windowWidth.value = window.innerWidth; }; onMounted(() => { window.addEventListener('resize', updateWidth); }); onUnmounted(() => { window.removeEventListener('resize', updateWidth); }); return { windowWidth, isMobile, isTablet, isDesktop }; }

🚀 Implementation Roadmap

Phase 1: Foundation (Week 1-2)

🎯 Goal: Establish Vue.js foundation with core functionality
  • Set up Vue 3 with Composition API
  • Implement Pinia store for state management
  • Convert core entry management functionality
  • Maintain existing localStorage persistence
  • Ensure PWA manifest and basic service worker work

Phase 2: Component Architecture (Week 3-4)

🎯 Goal: Break down into maintainable Vue components
  • Create component hierarchy (Header, Main, Footer, etc.)
  • Implement search functionality with Vue reactivity
  • Convert modals and notifications to Vue components
  • Add proper event handling and component communication
  • Implement Vue transitions for existing interactions

Phase 3: Enhancement & Polish (Week 5-6)

🎯 Goal: Add modern features while preserving minimalism
  • Implement glassmorphism design effects
  • Add micro-animations and improved transitions
  • Enhance accessibility with proper ARIA labels and focus management
  • Optimize for performance and mobile experience
  • Add comprehensive testing and documentation
⚠️ Migration Strategy: Consider running both versions in parallel during development. Create a migration tool to transfer localStorage data between versions if data structure changes are needed.

Success Criteria

  • Functional Parity: All existing features work identically
  • Performance: No degradation in load time or responsiveness
  • Mobile Experience: Improved touch interactions and gestures
  • Maintainability: Clean component structure for future updates
  • Accessibility: Meets WCAG 2.1 AA standards
  • Bundle Size: Minimize increase in total file size
Testing Strategy: Since this is a localStorage-based app, focus on manual testing across different browsers and devices. Consider implementing automated accessibility testing with tools like axe-core.
